FreeBSD 执行 portsnap fetch 时出错

想同步一下 ports。

也不知道从什么时候开始，执行 portsnap fetch update 时就报类似的错误：

/usr/sbin/portsnap: cannot open ca3d1528c2ecc4e2241979fac3ee3ce28eaxxxxxxxxxxxxxxxxxxx.gz: No such file or directory metadata is corrupt.

放狗搜了一下，找到解决方法，删除 /var/db/portsnap/files 目录：

rm -rf /var/db/portsnap/files

再执行 portsnap fetch update 完成后就不报错了。
